# 如何查看业务分析模块

## 快速开始

### 1. 启动开发服务器

```bash
npm run dev
```

等待服务器启动，你会看到类似这样的输出：
```
VITE v5.0.0  ready in 500 ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

### 2. 打开浏览器

访问 http://localhost:5173

### 3. 查看Dashboard

你应该能看到：
- 顶部标题："门店数据指标看板"
- 门店选择器
- 三个标签页：💰 财务分析、📊 业务分析、⚡ 动作分析

### 4. 点击"业务分析"标签

点击中间的"📊 业务分析"标签

### 5. 查看业务模块

点击后，你应该能看到：

#### 模块导航区域（白色背景）
10个业务模块标签，排列成网格：
- 📈 整体销售分析
- 🏷️ 市场销量与品牌
- 💼 主营分析
- 📡 运营商分析
- 👥 付费会员分析
- ♻️ 回收分析
- 📱 二手机分析
- ⌚ 智能产品分析
- 🔌 配件分析
- 🔧 维修分析

#### 内容区域（白色背景）
- 灰色调试信息框：显示当前模块ID
- 蓝色提示框：显示"✅ 整体销售分析模块已加载"
- 核心指标部分：4个KPI卡片
  - 总销售额：1,250,000元
  - 销售订单数：3,580单
  - 客单价：349元
  - 同比增长：15.2%
- 销售趋势图表
- 销售对比图表

### 6. 切换模块

点击任意其他模块标签，内容区域会切换到对应模块的内容。

## 如果看不到内容

### 检查清单

1. **确认服务器正在运行**
   ```bash
   # 检查是否有进程在运行
   lsof -i :5173
   ```

2. **清除浏览器缓存**
   - Chrome: Ctrl+Shift+Delete (Windows) 或 Cmd+Shift+Delete (Mac)
   - 选择"缓存的图片和文件"
   - 点击"清除数据"

3. **清除Vite缓存**
   ```bash
   rm -rf node_modules/.vite
   npm run dev
   ```

4. **检查浏览器控制台**
   - 按F12打开开发者工具
   - 切换到Console标签
   - 查看是否有错误信息（红色文字）

5. **查看调试日志**
   在控制台应该能看到：
   ```
   Rendering business module: overall_sales
   OverallSalesModule rendering
   ```

6. **检查DOM结构**
   - 按F12打开开发者工具
   - 切换到Elements标签
   - 搜索 `business-analysis-container`
   - 查看该元素是否存在且有内容

### 常见问题

#### 问题1: 点击业务分析后页面空白
**解决方案**:
1. 打开浏览器控制台查看错误
2. 刷新页面（Ctrl+R 或 Cmd+R）
3. 清除缓存后重试

#### 问题2: 看到模块标签但没有内容
**解决方案**:
1. 检查控制台是否有 `OverallSalesModule rendering` 日志
2. 使用Elements工具检查 `.business-module-content` 是否有内容
3. 检查CSS样式是否正确加载

#### 问题3: 模块标签不响应点击
**解决方案**:
1. 检查控制台是否有JavaScript错误
2. 刷新页面重试
3. 尝试使用无痕模式


应该看到所有项目都是 ✅

## 测试独立模块

如果Dashboard中看不到内容，可以测试模块是否能独立渲染：

1. 临时修改 `src/App.tsx`:
   ```typescript
   import { TestModules } from './test-modules';
   
   function App() {
     return <TestModules />;
   }
   
   export default App;
   ```

2. 保存文件，浏览器会自动刷新

3. 如果能看到"业务模块测试页面"和模块内容，说明模块本身没问题

4. 记得改回原来的代码：
   ```typescript
   import { DashboardWithFilters } from './components/DashboardWithFilters';
   
   function App() {
     return <DashboardWithFilters />;
   }
   
   export default App;
   ```
