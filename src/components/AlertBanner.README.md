# AlertBanner Component

## Overview

The AlertBanner component displays alerts at the top of the dashboard, providing users with immediate visibility into metrics that are below their target values.

## Requirements

- **Requirement 10.4**: WHEN 实际指标值低于目标值时，THE System SHALL 标记为未达标状态

## Features

- **Alert Count Summary**: Shows total number of alerts and breakdown by severity
- **Severity Levels**: Displays alerts with four severity levels (INFO, WARNING, ERROR, CRITICAL)
- **Top Alerts Display**: Shows the 3 most critical alerts by default
- **Expandable List**: Users can expand to view all alerts
- **Dismissible**: Individual alerts can be dismissed
- **View Details**: Click to view detailed information about each alert

## Usage

```tsx
import { AlertBanner } from './components/AlertBanner';
import { Alert } from './types';

function Dashboard() {
  const [alerts, setAlerts] = useState<Alert[]>([]);

  const handleDismiss = (alertId: string) => {
    setAlerts(prev => prev.filter(a => a.alertId !== alertId));
  };

  const handleViewDetails = (alert: Alert) => {
    // Open alert details modal
  };

  return (
    <AlertBanner
      alerts={alerts}
      onDismiss={handleDismiss}
      onViewDetails={handleViewDetails}
    />
  );
}
```

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `alerts` | `Alert[]` | Yes | Array of alert objects to display |
| `onDismiss` | `(alertId: string) => void` | No | Callback when an alert is dismissed |
| `onViewDetails` | `(alert: Alert) => void` | No | Callback when "详情" button is clicked |

## Alert Severity

Alerts are categorized by severity based on how far below target the metric is:

- **CRITICAL** (🔴): >30% below target
- **ERROR** (⚠️): 15-30% below target
- **WARNING** (⚡): 5-15% below target
- **INFO** (ℹ️): 1-5% below target

## Behavior

1. **No Alerts**: Component returns null and doesn't render anything
2. **Default View**: Shows top 3 most critical alerts
3. **Expanded View**: Shows all alerts when toggle button is clicked
4. **Dismissed Alerts**: Removed from view but can be restored by refreshing data
5. **Sorting**: Alerts are automatically sorted by severity (CRITICAL first)
